● Personal AI Project

Personal AI Council:
Hub-and-Spoke Agent System

A personal multi-agent system built with a mix of local Ollama models and API-routed calls to Claude, running on a 16GB M4 Mac mini. Tyler sets the vision. Zeus runs the show.

A persistent, multi-agent AI council where Zeus ⚡ (ChatGPT 5.4) is the central hub: routing tasks, owning the queue, and enforcing governance. Three advisors (Prometheus, Argus, Hestia) consult but never command. Five execution agents receive orders from Zeus only. No agent ever talks to another agent. All routing through Zeus. All goals set by Tyler.

Agents never communicate directly — all routing goes through Zeus

Architecture
Why Hub-and-Spoke?

Zeus is the only node with a full picture of system state. Advisors (Prometheus, Argus, Hestia) inform but never command. Execution agents (Atlas, Muse, Hermes, Hephaestus, Plutus) receive one task, execute it in isolation, and return a result. This eliminates trust chains, makes Argus's audit logging trivial, and means a hallucinating Muse or hung Hermes can only damage its own task scope.

Why Zeus is Critical

Without a single point of orchestration, agents accumulate state, contradict each other, and spiral into recursive loops. Zeus enforces sequencing, holds the approval gates, owns escalation to Tyler, and is the only node that can write to Prometheus's memory store. Remove Zeus and the advisors become noise while the execution agents become uncoordinated tools.

How Agents Connect: MCP

MCP replaces custom integrations. Every tool is a standardized server the agent calls the same way.

Security Design
🔒 Standard Profile, No Sudo

The entire agent system runs on a dedicated standard macOS user profile with no administrator rights. Even if an agent is compromised or produces a malicious shell command, it cannot escalate privileges, install software system-wide, or touch system files. Sudo is physically unavailable to this profile.

🧱 Memory Partition

Agent memory is sandboxed to a dedicated directory within the standard profile's home folder. No agent has a path reference to any other user's home directory, system keychain, or sensitive config files. Prometheus reads and writes only within this bounded store. Cross-user reads are structurally impossible.

🚫 No Sideways Communication

Execution agents have no knowledge of each other's existence. They receive a task payload from Zeus, operate within their MCP tool scope, and return a result. There is no shared message bus, no direct agent-to-agent API, and no shared writable state outside Prometheus's memory store, which only Zeus can write to.

✅ Argus Reviews Every Action

Before Zeus dispatches any execution agent task involving files, shell, external calls, or outbound messages, Argus classifies the risk. High-risk or ambiguous actions are escalated to Tyler via Discord rather than proceeding. The audit log is append-only and written to a path that execution agents cannot access.

Technical Decisions
Model Stratification: Local, Deepseek, and API

Every task routes to the minimum-capable model that solves it reliably. Advisors (Prometheus, Argus, Hestia) run local Ollama or Deepseek models: free, private, and fast for classification tasks. Execution agents use Deepseek R1 for reasoning-heavy work (cost-efficient), Claude Haiku for lightweight retrieval, and Sonnet/Opus for creative or high-stakes decisions. This stretches API budget and keeps latency low for local work.

# Zeus task routing with model selection zeus.route(task): argus.review(task) # Ollama/Deepseek local hestia.check() # no model needed if memory_budget < 4GB: use_local = true select_model(task): if task.needs_reasoning: return deepseek_r1 # cost-efficient thinking elif task.is_retrieval: return haiku # fast API call elif task.is_creative: return sonnet # quality matters
The 16GB Constraint

Running multiple model processes on 16GB of unified memory means memory pressure is a first-class design constraint. Baseline wired memory sits around 1GB. Loading Ollama alongside a Claude Code session and the Openclaw daemon pushes toward swap. Agents run serially through Zeus, not in parallel. This turns out to be the correct governance pattern regardless of hardware.

Why Logging Matters

Without an audit trail, reconstructing why Zeus dispatched a specific agent with a specific payload is guesswork. Argus writes a structured log entry for every task: timestamp, agent, task type, risk classification, and outcome. This turned debugging from guesswork into forensics.
